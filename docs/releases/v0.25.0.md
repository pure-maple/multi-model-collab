# modelmux v0.25.0 — Security Hardening Release

**Release Date**: 2026-03-07
**Tests**: 361 passed

## Highlights

This release addresses all findings from a comprehensive 3-agent parallel security audit of the entire codebase (server.py, adapters, A2A HTTP server, dashboard, notifications, policy engine).

## Security Fixes

### Critical
- **SSRF Protection** (a2a/http_server.py): Push notification URLs are now validated against private/reserved IP ranges (RFC 1918), loopback addresses, link-local (169.254.x.x), and cloud metadata endpoints. Blocks `file://`, `ftp://`, and other non-HTTP schemes.
- **A2A Policy Bypass** (a2a/http_server.py): The A2A HTTP server now enforces the same policy engine rules (provider allowlist/blocklist, sandbox restrictions, rate limits) as the MCP path. Previously, the entire policy layer was completely bypassed on the A2A route.

### High
- **Subprocess Zombie Prevention** (adapters/base.py): `process.terminate()` now has a `process.kill()` fallback when graceful shutdown times out, preventing zombie processes.

### Medium
- **Sandbox Escalation** (adapters/codex.py): Unknown sandbox values now default to `"read-only"` instead of being passed through to the CLI unchanged.
- **CLI Flag Injection** (adapters/base.py): `sanitize_extra_args()` strips any `extra_args` values starting with `-` to prevent argument injection via model/profile parameters.
- **Dashboard Input Validation** (dashboard.py): All query parameters (`limit`, `hours`, `bucket`) are clamped to safe ranges with `_clamp_int()`/`_clamp_float()`.

## Infrastructure

- **Structured Logging** (`log.py`): New centralized logging module
  - `MODELMUX_LOG_LEVEL` env var: `DEBUG` / `INFO` / `WARNING` / `ERROR`
  - `MODELMUX_LOG_FORMAT` env var: `text` (human-readable) or `json` (structured)
  - Integrated at both CLI and MCP server entry points
  - All child loggers (`modelmux.a2a.http`, `modelmux.notifications`, etc.) inherit configuration

## Test Coverage

48 new tests:
- 30 security tests (SSRF, sandbox, flag injection, policy enforcement)
- 10 logging tests (level config, env override, JSON format, idempotency)
- 8 dashboard param validation tests

# modelmux v0.25.0 — 安全加固版

**发布日期**: 2026-03-07
**测试**: 361 通过

## 核心变更

本版本修复了通过 3 个 agent 并行安全审计发现的所有问题，覆盖 server.py、适配器、A2A HTTP 服务器、Dashboard、通知系统和策略引擎。

## 安全修复

- **SSRF 防护**: Push 通知 URL 验证（阻止内网/回环/云元数据端点）
- **A2A 策略绕过**: A2A HTTP 路径现在执行与 MCP 路径相同的策略检查
- **僵尸进程防护**: subprocess 超时后的 kill 回退
- **Sandbox 提权修复**: 未知 sandbox 值默认降级为 read-only
- **CLI Flag 注入防护**: 拒绝以 `-` 开头的 extra_args 值
- **Dashboard 输入验证**: 查询参数边界检查

## 基础设施

- **结构化日志**: `MODELMUX_LOG_LEVEL` / `MODELMUX_LOG_FORMAT` 环境变量配置
